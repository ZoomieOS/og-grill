# OG Original Grill — лендинг (Preact + Vite + i18n)

Лендинг ресторана грузинской кухни **OG Original Grill** (Варшава) на Preact + Vite,
с мультиязычностью (PL по умолчанию, плюс EN / GE / RU) и деплоем на **Vercel**.

## Структура проекта

```
├── index.html                # точка входа Vite (дефолтные PL title/description)
├── vercel.json                 # конфиг сборки и SPA-rewrite для Vercel
├── vite.config.js
├── package.json
└── src/
    ├── main.jsx                # render(<App />)
    ├── App.jsx                  # композиция страницы + LanguageProvider + scroll-reveal
    ├── style.css                # все стили (фирменная палитра OG)
    ├── i18n/
    │   ├── translations.js      # словари PL / EN / GE / RU
    │   └── LanguageContext.jsx  # контекст языка, дефолт PL, сохранение в localStorage
    └── components/
        ├── Logo.jsx
        ├── ClipDefs.jsx         # фирменная "огненная" SVG-рамка (clip-path)
        ├── Header.jsx
        ├── LanguageSwitcher.jsx # кнопки PL / EN / GE / RU
        ├── Hero.jsx
        ├── Marquee.jsx
        ├── About.jsx
        ├── Highlights.jsx       # карточки меню/сервиса (как Instagram-хайлайты)
        ├── Gallery.jsx
        ├── Info.jsx              # часы, адрес, контакты
        └── Footer.jsx
```

## Локализация (i18n)

- Дефолтный язык — **PL** (польский), как у самого бренда в Варшаве.
- Дополнительно доступны **EN**, **GE** (ქართული) и **RU**.
- Переключатель — в шапке и в футере (кнопки PL/EN/GE/RU), активный язык подсвечен красным.
- Выбор языка сохраняется в `localStorage` — при повторном визите подхватится тот же язык.
- `<title>` и meta description тоже переключаются на лету (см. `LanguageContext.jsx`).
- Все тексты лежат в одном файле `src/i18n/translations.js` — структура для всех 4 языков
  одинаковая (проверено скриптом при сборке), так что добавить ещё один язык — это просто
  скопировать блок `pl {...}` и перевести значения.

> Грузинский перевод сделан с опорой на стандартную терминологию (например, шашлык — это
> аутентичное «მწვადი», а не транслитерация), но для финальной публикации рекомендую отдать
> копирайт на вычитку носителю языка.

## Деплой на Vercel

### Вариант A — через Vercel Dashboard + GitHub (рекомендуется)

1. Запушьте проект в репозиторий на GitHub:
   ```bash
   git init
   git add .
   git commit -m "OG Original Grill landing"
   git branch -M main
   git remote add origin <ваш-репозиторий>
   git push -u origin main
   ```
2. На [vercel.com](https://vercel.com): **Add New → Project** → выберите репозиторий.
3. Vercel сам определит фреймворк как **Vite** (это также явно зафиксировано в `vercel.json`):
   `npm install` → `npm run build` → раздача папки `dist/`.
4. Никаких переменных окружения не требуется — сайт полностью статический.
5. После деплоя получите публичный URL вида `og-original-grill.vercel.app`.

### Вариант B — через Vercel CLI

```bash
npm i -g vercel
vercel login
vercel        # деплой preview-версии
vercel --prod # деплой в продакшен
```

CLI сам подхватит настройки из `vercel.json`.

## SEO

Что уже настроено в проекте:

- **Title / meta description** — на польском по умолчанию, переключаются на лету через
  `LanguageContext.jsx` при смене языка (для уже открытой страницы — полезно для UX, но
  поисковики видят только то, что отдаётся при первой загрузке, см. ограничение ниже).
- **Open Graph + Twitter Card** — `og:title`, `og:description`, `og:image` (1200×630,
  сгенерирована в фирменном стиле, лежит в `public/og-image.jpg`), `og:locale` +
  `og:locale:alternate` для всех 4 языков. Это то, что показывается при шаринге ссылки в
  WhatsApp/Telegram/Facebook/X.
- **JSON-LD структурированные данные** (`Restaurant` / `LocalBusiness`) в `index.html` — название,
  адрес, кухня, часы работы, ссылка на Instagram. Помогает Google показывать карточку ресторана
  (часы, адрес) прямо в выдаче.
- **robots.txt + sitemap.xml** в `public/` — разрешают индексацию и подсказывают поисковику
  единственный URL сайта.
- **canonical** ссылка на основной URL.
- **Иконки**: `favicon.svg` (векторная, фирменный флейм-знак), `favicon.ico`/`favicon-32x32.png`
  (для старых браузеров и поисковых систем, которые ещё не умеют SVG-фавиконки),
  `apple-touch-icon.png` (для iOS «На экран Домой»).
- **Исправлен языковой код**: внутри проекта язык называется `ge` (как в био бренда в Instagram:
  `PL · EN · GE · RU`), но настоящий ISO 639-1 код грузинского языка — `ka` (`GE` — это код
  страны Грузия по ISO 3166, а не языка). Атрибут `<html lang>` и `og:locale` теперь используют
  правильный `ka`/`ka_GE` — это важно и для SEO, и для скринридеров.

### ⚠️ Перед деплоем замените домен

Во всех файлах (`index.html`, `public/robots.txt`, `public/sitemap.xml`) сейчас стоит
плейсхолдер `https://og-original-grill.vercel.app/`. После первого деплоя на Vercel замените
его на настоящий адрес сайта (свой поддомен Vercel или кастомный домен) везде, где он
встречается — иначе canonical/OG-теги будут указывать в никуда.

### Важное ограничение архитектуры (SPA)

Сайт — это одна HTML-страница, языки переключаются на клиенте (JS), а не отдельными URL
(`/en`, `/ru`, `/ka`). Из этого следует:

- Поисковики (в первую очередь Google) **видят только польскую версию** при обходе — именно
  её отдаёт сервер при первой загрузке. Контент на EN/GE/RU физически не проиндексируется как
  отдельная страница, потому что у него нет своего URL.
- Современный Googlebot умеет выполнять JS и в целом неплохо индексирует SPA, но это медленнее
  и менее надёжно, чем обычный HTML — особенно для других систем (Bing, соцсети, мессенджеры),
  которые JS не выполняют вовсе.
- Для полноценного мультиязычного SEO (с `hreflang`, отдельными `sitemap`-записями и
  индексацией каждого языка как отдельной страницы) нужны отдельные URL — например,
  `/`, `/en`, `/ka`, `/ru` — с пререндером HTML под каждый язык на этапе сборки.

Если это важно для бизнеса (например, если ожидается органический трафик из англоязычного или
русскоязычного поиска, а не только переход по ссылке из Instagram) — это отдельная,
более крупная доработка: маршрутизация + статическая генерация HTML на каждый язык +
`hreflang`-теги + расширенный `sitemap.xml`. Текущая версия — это «один язык для Google
(польский), остальные — удобство для тех, кто уже зашёл на сайт».

  `preact-iso` или `wouter`) с языковым префиксом в пути.
